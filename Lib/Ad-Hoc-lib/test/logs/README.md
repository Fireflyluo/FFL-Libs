# logs — 仿真运行日志

每次仿真运行产生的节点日志文件存放在此目录。文件受 `.gitignore` 保护, 不会进入版本控制。

## 文件命名规则

```
node_{节点索引}_{节点ID}.log
```

| 部分 | 含义 | 示例 |
|------|------|------|
| `node_` | 固定前缀 | `node_` |
| `{索引}` | 仿真内部分配的 0-based 索引 | `0`, `1`, `5` |
| `{节点ID}` | 协议层 node_id | `1` (网关), `1000001` (信标) |

例如: `node_0_1.log` = 索引0, 节点ID=1 (网关)

## 日志内容格式

```
[HH:MM:SS.ms] [NODE:节点ID] 事件描述
```

### 事件类型

| 事件 | 含义 |
|------|------|
| `INIT: role=...` | 节点初始化 |
| `STATE: ST1/U1/C1/UN/CN` | 状态转换 |
| `JOINED: level=...` | 入网成功 |
| `TX class=A/D level=...` | 发送帧 |
| `RX class=A/D level=... rssi=...` | 接收帧 |
| `DATA SUBMIT: seq=...` | 提交业务数据 |
| `DATA ACKED: src=... lmt_d=...` | 数据被确认 |
| `DATA RETRY_EXHAUSTED: ...` | 数据重试耗尽 |
| `NETWORK_LOCKED` | 网关网络窗口关闭 |

## 场景对应关系

| 最后修改时间 | 对应场景 | 节点规模 |
|-------------|----------|---------|
| 最近一次运行 | 当前 SCENARIO_MODE | 取决于编译宏 |

每次重新运行仿真, 日志文件会被截断覆盖。
